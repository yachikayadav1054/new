Dataverse Fundamentals
▼
What is Microsoft Dataverse?
A cloud-based, low-code data platform built on Azure that stores and manages business data used by Power Platform and Dynamics 365. It provides built-in security, business logic, and integration capabilities.
Dataverse was previously called Common Data Service (CDS). Mention this transition to show awareness.

What is a Dataverse Environment?
An isolated container holding data, apps, flows, and customizations. Each org gets a Default environment; additional environments (Sandbox, Production, Developer) can be created per business need.
Don't confuse "Environment" with "Organization" — in Dataverse, each environment has exactly one database (org). Classic trap question.
What is the Dataverse Web API?
A RESTful API based on OData v4 protocol. Used to perform CRUD operations, execute functions/actions, and query data.
GET /api/data/v9.2/accounts?$select=name,revenue&$filter=revenue gt 100000

SystemPlatform tables (User, Team, Role) — limited customization
Owner vs Organization table ownership
  • User/Team owned: Each record has an owner. Security model applies sharing/team-based access.
  • Organization owned: All records belong to the org; any user with table access sees all records.
Currency table = Organization owned. Account = User/Team owned.
You CANNOT change table ownership type after creation. Classic interview trap!

What is a Polymorphic Lookup?
A lookup column that can reference multiple table types. Called "Customer" (Account or Contact) or "Regarding" (any activity-enabled table). Internally uses the activityparty pattern.

Parental behavior means child records follow parent's lifecycle. Use Restrict Delete to prevent accidental deletion of parent records that have children.
